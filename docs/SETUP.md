# Setup Guide

## Prerequisites

- Claude Code installed and working
- Git installed
- ~5 minutes

---

## Installation

### Step 1: Clone Harness Skills Repository

```bash
git clone https://github.com/chetan25/harness-claude-skills.git
```

This gives you the templates and core skills needed.

---

### Step 2: Copy `.claude/` to Your Project

For an existing project:

```bash
cp -r harness-claude-skills/.claude/skills/ your-project/.claude/
```

Your project now has:
```
your-project/
├── .claude/
│   └── skills/
│       └── README.md              ← Guide for domain skills
├── src/
├── tests/
└── ...
```

---

### Step 3: Open in Claude Code

```bash
cd your-project
# Open with Claude Code
```

---

### Step 4: First Prompt (Project Analysis)

Say to Claude Code:

```
"Analyze this project structure and create domain skills. 
I want you to understand my codebase, patterns, and conventions 
so that when I ask for features, you generate code that fits."
```

Claude will:

1. ✅ Scan your project (src/, tests/, package.json, etc.)
2. ✅ Detect technology stack (React, Node, Python, etc.)
3. ✅ Extract coding patterns
4. ✅ Analyze architecture
5. ✅ Create `claude.md` at project root
6. ✅ Generate `.claude/skills/`:
   - `patterns-{lang}.md` — Your coding style
   - `architecture.md` — Project structure
   - `design-tokens.md` — UI tokens (if found)
   - `test-patterns.md` — How you test
   - `api-contracts.md` — Public interfaces (if found)

---

### Step 5: Start Using Harness

Now just ask Claude for features naturally:

```
"Add user authentication with email/password login and JWT tokens"
```

or

```
"Create a dark mode toggle that persists to localStorage"
```

Claude will automatically:
- Load context from `.claude/skills/`
- Decompose into tasks
- Generate code matching your patterns
- Write tests
- Verify everything works

## Configuration

### Customize Domain Skills

Edit `.claude/skills/patterns-*.md` to guide Claude:

```markdown
# React Patterns in This Project

## Component Structure
- All components in `src/components/`
- One component per file
- Named exports + default export
- TypeScript interfaces required

## Hooks Pattern
- Custom hooks in `src/hooks/`
- Prefix with `use` (e.g., `useAuth`)
- Document with JSDoc

## Testing
- Unit tests: `__tests__/` folder
- Use React Testing Library
- Minimum 80% coverage
```

---

### Add MCPs (Figma, APIs, etc.)

Edit `.claude/mcp-config.yaml`:

```yaml
mcps:
  figma:
    enabled: true
    token: ${FIGMA_API_TOKEN}
    project_id: your-figma-project-id
  
  # Add custom MCPs
  custom_api:
    enabled: true
    endpoint: https://api.example.com
    auth: bearer ${API_TOKEN}
```

Then Claude can:
- Fetch Figma designs
- Validate UI against design specs
- Call your APIs
- Reference API documentation

---

### Orchestrator Configuration

Create `.claude/orchestrator.md`:

```markdown
# Task Orchestration for [Project Name]

## Phases
1. DECOMPOSE: Break into tasks
2. CONTEXT: Load patterns + architecture
3. CODE: Generate with grounding
4. VERIFY: Tests, linting, types
5. UI_CHECK: Figma comparison (if needed)
6. TEST: Write & run tests
7. ITERATE: Refine until done

## Quality Gates
- All tests pass
- ESLint: no errors
- TypeScript: no errors
- Test coverage: ≥ 80%
- Figma match: ≥ 90% (if UI task)

## Context Sources
- Patterns: .claude/skills/patterns-*.md
- Architecture: .claude/skills/architecture.md
- Design: .claude/skills/design-tokens.md
- Tests: .claude/skills/test-patterns.md
```

## Verification

After setup, verify everything works:

1. **Check `.claude/` exists:**
   ```bash
   ls -la .claude/skills/
   ```
   Should see: `patterns-*.md`, `architecture.md`, `design-tokens.md`, etc.

2. **Check `claude.md` at project root:**
   ```bash
   cat claude.md | head -20
   ```
   Should mention "harness-orchestrator" and skills location.

3. **Test with a small feature:**
   - Say: "Add a welcome message to the homepage"
   - Claude should generate code matching your patterns
   - Tests should pass

## Troubleshooting

### Skills not being used
- Verify `.claude/skills/` exists and has `.md` files
- Verify `claude.md` mentions `.claude/skills/`
- Say "Reanalyze this project" to Claude

### Generated code doesn't match style
- Run analysis again: "Re-analyze this project"
- Manually update `.claude/skills/patterns-*.md`
- Ask Claude: "Update patterns based on recent code"

### Claude forgets context between prompts
- That's normal! `claude.md` reloads skills each time
- Each prompt gets fresh context from `.claude/skills/`

### Tests keep failing
- Check `.claude/skills/test-patterns.md` is accurate
- Update test setup if needed
- Ask Claude: "Fix the test setup for [framework]"
